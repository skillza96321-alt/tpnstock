# คลังสินค้า — ระบบสต็อกสินค้า (PWA + Google ชีตเป็นฐานข้อมูลกลาง)

แอปนี้เป็น **Progressive Web App (PWA)** ติดตั้งลงหน้าจอโฮมได้ทั้ง iPhone และ Android
และใช้ **Google ชีต** เป็นฐานข้อมูลกลาง ทำให้ทุกคนที่เข้าใช้งานเห็นข้อมูลสต็อกชุดเดียวกัน
แถมเปิดดู/แก้ไขข้อมูลดิบใน Google ชีตได้โดยตรงด้วยตัวเอง

> ทำไมต้องมี Apps Script ด้วย: เบราว์เซอร์เชื่อมต่อเขียนข้อมูลลง Google ชีตตรงๆ ไม่ได้
> ต้องมีตัวกลางที่เรียกว่า **Google Apps Script** ทำหน้าที่รับคำขอจากแอป แล้วไปเขียน/อ่านชีตแทน
> (เป็นวิธีมาตรฐานที่ใช้กันทั่วไปเวลาจะให้ Google ชีตทำหน้าที่เป็น "ฐานข้อมูล" ของเว็บแอป)

## ขั้นตอนที่ 1: สร้าง Google ชีต

1. ไปที่ https://sheets.google.com สร้างชีตใหม่ ตั้งชื่ออะไรก็ได้ เช่น "ฐานข้อมูลคลังสินค้า"
2. สร้างแท็บ (sheet tab) ด้านล่าง 3 แท็บ ตั้งชื่อให้ตรงตัวพิมพ์เล็กทั้งหมดดังนี้:
   - `categories`
   - `products`
   - `transactions`
   (คลิกขวาที่แท็บ "Sheet1" เดิม เพื่อเปลี่ยนชื่อ/เพิ่มแท็บใหม่)
3. ไม่ต้องพิมพ์หัวตารางเอง — ปล่อยทุกแท็บว่างไว้ก่อนได้เลย ระบบจะใส่หัวตารางภาษาไทยให้อัตโนมัติในขั้นตอนที่ 2.5

## ขั้นตอนที่ 2: ติดตั้ง Apps Script

1. ในหน้า Google ชีต ไปที่เมนู **Extensions (ส่วนขยาย) → Apps Script**
2. จะเจอไฟล์ `Code.gs` เปิดอยู่ ให้**ลบโค้ดเดิมทั้งหมด**ในนั้น
3. เปิดไฟล์ **AppsScript.gs** ที่แนบมาด้วย คัดลอกทั้งหมดไปวางแทน
4. กด **Save** (ไอคอนรูปแผ่นดิสก์ หรือ Ctrl+S)
5. กด **Deploy → New deployment**
6. กดไอคอนเฟือง ⚙️ ข้างคำว่า "Select type" เลือก **Web app**
7. ตั้งค่า:
   - Execute as: **Me**
   - Who has access: **Anyone**
8. กด **Deploy**
9. จะมีหน้าต่างขอสิทธิ์ (Authorize access) — กด **Authorize access** แล้วเลือกบัญชี Google ของคุณ
   ถ้าเจอหน้าเตือน "Google hasn't verified this app" ให้กด **Advanced** → **Go to (ชื่อโปรเจกต์) (unsafe)** → **Allow**
   (ปลอดภัย เพราะเป็นสคริปต์ที่คุณเขียน/วางเอง ไม่ใช่แอปจากคนแปลกหน้า)
10. คัดลอก **Web app URL** ที่ได้ (ขึ้นต้นด้วย `https://script.google.com/macros/s/...`) เก็บไว้

## ขั้นตอนที่ 2.5: ใส่หัวตารางภาษาไทย + จัดรูปแบบให้สวยงาม (ทำครั้งเดียว แนะนำให้ทำ)

หลังวางโค้ดเสร็จ ให้**ปิดหน้าต่าง Apps Script แล้วกลับไปที่หน้า Google ชีต แล้วรีเฟรชหน้าเว็บ (F5)**
จะเห็นเมนูใหม่ขึ้นที่แถบเมนูด้านบนชื่อ **"⚙️ คลังสินค้า"**

1. กดเมนู **⚙️ คลังสินค้า → จัดรูปแบบชีตให้สวยงาม (ภาษาไทย)**
2. ครั้งแรกจะมีหน้าต่างขอสิทธิ์อีกครั้ง กด **ดำเนินการต่อ/Continue** แล้วอนุญาตเหมือนขั้นตอนก่อนหน้า
3. ระบบจะ**ใส่หัวตารางเป็นภาษาไทยให้อัตโนมัติ** เช่น "ชื่อสินค้า", "รหัสหมวดหมู่", "จำนวน" ฯลฯ
   พร้อมจัดหัวตารางให้เป็นสีเขียวตัวหนาตรึงไว้ด้านบน ใส่แถบสีสลับแถวให้อ่านง่าย
   ปรับความกว้างคอลัมน์ให้พอดี และทำ dropdown ให้เลือกได้แค่ `in`/`out` ในคอลัมน์ประเภทของแท็บ transactions
4. วางเมาส์ชี้ที่หัวตารางแต่ละช่อง จะมีคำอธิบายเล็กๆ (note) บอกความหมายและชื่อฟิลด์ภายในกำกับไว้

จัดรูปแบบแล้วไม่กระทบข้อมูลใดๆ ทั้งสิ้น รันซ้ำได้เรื่อยๆ ถ้าอยากจัดใหม่

> **หมายเหตุสำคัญ:** ตอนนี้ระบบอ่าน/เขียนข้อมูลโดยอิง**ลำดับคอลัมน์** ไม่ได้อิงจากชื่อหัวตารางอีกต่อไป
> ดังนั้นจะพิมพ์หัวตารางเป็นภาษาไทยหรือภาษาอะไรก็ได้ตามใจชอบ แต่**ห้ามสลับลำดับคอลัมน์ หรือแทรก/ลบคอลัมน์**ในแท็บทั้ง 3 นี้เด็ดขาด ไม่งั้นข้อมูลจะเพี้ยน (จะเพิ่ม/ลบ**แถว**ข้อมูลได้ตามปกติ ไม่มีปัญหา)

## ขั้นตอนที่ 3: ใส่ค่าเชื่อมต่อในแอป

1. เปิดไฟล์ **index.html** ด้วยโปรแกรมแก้ไขข้อความ (เช่น Notepad, VS Code)
2. หาบรรทัดนี้ใกล้ๆ ต้นไฟล์ (กด Ctrl+F ค้นคำว่า `YOUR_GOOGLE_SCRIPT_URL`):
   ```js
   const GAS_URL = 'YOUR_GOOGLE_SCRIPT_URL';
   ```
3. แทนที่ด้วยลิงก์ Web app URL จากขั้นตอนที่ 2 เช่น:
   ```js
   const GAS_URL = 'https://script.google.com/macros/s/AKfycb.../exec';
   ```
4. บันทึกไฟล์

## ขั้นตอนที่ 4: อัปโหลดขึ้นโฮสต์ฟรี

**Netlify Drop (ง่ายที่สุด):**
1. เข้า https://app.netlify.com/drop
2. ลากทั้งโฟลเดอร์นี้ (index.html, manifest.json, icons/ ฯลฯ) ไปวาง
3. ถ้าเป็นเว็บเดิมที่เคยอัปโหลดไว้ (เช่น tpnstock.netlify.app) ให้ลากไฟล์ชุดใหม่ไปวางทับที่หน้า site เดิมใน Netlify dashboard ได้เลย

## ขั้นตอนที่ 5: ติดตั้งลงมือถือ

**iPhone (Safari เท่านั้น):** เปิดลิงก์ → กดปุ่มแชร์ → "เพิ่มไปยังหน้าจอโฮม" → กด "เพิ่ม"
**Android (Chrome):** เปิดลิงก์ → เมนู 3 จุด → "ติดตั้งแอป" → กด "ติดตั้ง"

ทุกคนที่ติดตั้งแอปจากลิงก์เดียวกันนี้ จะเห็น**ข้อมูลสต็อกชุดเดียวกัน**
(แอปจะรีเฟรชข้อมูลอัตโนมัติทุก 20 วินาที หรือกดปุ่ม "รีเฟรชข้อมูล" ที่หน้าภาพรวมเพื่อดูข้อมูลล่าสุดทันที)

## ปิดรอบบัญชีประจำเดือนอัตโนมัติ (ไม่บังคับ)

ถ้าอยากให้ระบบล้างรายการรับ-จ่ายทุกสิ้นเดือนอัตโนมัติ แต่ยังคงยอดสต็อกปัจจุบันไว้ (ไม่รีเซ็ตยอดเป็น 0)
ทำได้ครับ กลไกคือ: ทุกสิ้นเดือนระบบจะ **ยุบยอดสต็อกปัจจุบันให้กลายเป็น "ยอดตั้งต้น" ของเดือนถัดไป**
แล้ว**ย้ายรายการรับ-จ่ายทั้งหมดของเดือนนั้นไปเก็บสำรองไว้ในแท็บใหม่ชื่อ `transactions_archive`** (ไม่ได้ลบทิ้งถาวร ย้อนดูประวัติได้เสมอ)
จากนั้นล้างแท็บ `transactions` ให้ว่างพร้อมเริ่มรอบใหม่

**เปิดใช้งาน (ทำครั้งเดียว):**
1. อัปเดตโค้ด `AppsScript.gs` เป็นเวอร์ชันล่าสุด (วางทับใน Apps Script แล้ว Save)
2. กลับไปหน้าชีต รีเฟรชหน้าเว็บ (F5)
3. กดเมนู **⚙️ คลังสินค้า → เปิดใช้งานปิดรอบอัตโนมัติทุกสิ้นเดือน**
4. อนุญาตสิทธิ์เพิ่มเติมที่ขอ (ต้องใช้สิทธิ์ตั้งเวลาทำงานอัตโนมัติ และส่งอีเมลสรุปผล)

จากนั้นระบบจะตรวจสอบทุกคืนประมาณ 23:00 น. และปิดรอบให้อัตโนมัติเฉพาะคืนสุดท้ายของแต่ละเดือนเท่านั้น (ทำวนซ้ำทุกเดือนตลอด 12 เดือน ไม่ต้องตั้งใหม่รายเดือน)
เมื่อปิดรอบเสร็จจะมีอีเมลสรุปส่งไปที่บัญชี Google ของคุณอัตโนมัติทุกครั้ง

**ทดสอบ/สั่งปิดรอบด้วยมือได้ทุกเมื่อ:** กดเมนู **⚙️ คลังสินค้า → ปิดรอบบัญชีเดือนนี้เลยตอนนี้ (มือ)**
จะมีหน้าต่างถามยืนยันก่อนทุกครั้ง (การกระทำนี้ย้อนกลับไม่ได้ทันที แต่ข้อมูลยังอยู่ครบใน `transactions_archive`)

## ⚠️ ข้อควรรู้เกี่ยวกับการใช้ Google ชีตเป็นฐานข้อมูล- **ไม่มีระบบล็อกอิน/รหัสผ่าน** ใครก็ตามที่มีลิงก์เว็บแอปสามารถเพิ่ม/แก้ไข/ลบข้อมูลได้ทั้งหมด
  เหมาะสำหรับใช้งานภายในทีมที่ไว้ใจกัน
- **ห้ามสลับลำดับคอลัมน์ หรือแทรก/ลบคอลัมน์ หรือลบ/สลับชื่อแท็บเอง** ไม่งั้นแอปจะอ่านข้อมูลผิดพลาด
  (แก้ข้อความหัวตาราง แก้ไขข้อมูลในแถวอื่น หรือเพิ่ม/ลบแถวข้อมูล ทำได้ตามปกติ ไม่มีปัญหา)
- **รูปสินค้าจะถูกบีบอัดให้เล็กลงมาก** ก่อนเก็บ เพราะ Google ชีตจำกัดตัวอักษรต่อ 1 ช่องไว้ที่ 50,000 ตัวอักษร
  ถ้ารูปมีรายละเอียดซับซ้อนมากอาจถูกปฏิเสธ ให้ลองรูปที่เรียบง่ายกว่า
- ความเร็วในการโหลด/บันทึกข้อมูลจะช้ากว่าฐานข้อมูลจริงเล็กน้อย (ปกติ 1-2 วินาทีต่อครั้ง)
  เหมาะกับทีมขนาดเล็ก-กลาง ไม่เหมาะกับการใช้งานพร้อมกันจำนวนมากๆ
- ระหว่างบันทึก/ลบข้อมูล ปุ่มจะเปลี่ยนเป็น "กำลังบันทึก..." และกดซ้ำไม่ได้จนกว่าจะเสร็จ
  เพื่อป้องกันไม่ให้เกิดรายการซ้ำจากการกดรัวๆ ระหว่างรอเชื่อมต่อ Google ชีต — ถ้ากดครั้งเดียวแล้วรอ
  แม้จะดูช้าไปหน่อย ข้อมูลจะถูกบันทึกแค่ครั้งเดียวแน่นอน
- Google Apps Script แบบฟรีมีโควตาการเรียกใช้งานราว 20,000 ครั้ง/วัน ซึ่งเพียงพอสำหรับการใช้งานทั่วไป

## โครงสร้างไฟล์
```
index.html          → ตัวแอปหลัก (มีค่า GAS_URL ให้กรอกที่ต้นไฟล์)
AppsScript.gs        → โค้ดสำหรับวางใน Google Apps Script (ทำหน้าที่เชื่อมชีตกับแอป)
manifest.json        → ข้อมูลแอปสำหรับติดตั้งลงมือถือ
service-worker.js    → ทำให้เปิดแอปได้แม้เน็ตหลุดชั่วคราว (ข้อมูลยังต้องใช้เน็ตในการซิงก์)
icons/                → ไอคอนแอป
```
